### Interface Definition

The `IObservabilityClient` interface allows you to integrate with your own monitoring and observability systems:

```go
type IObservabilityClient interface {
    // Initialize the observability client
    Init(ctx context.Context) error
    
    // Report counter metrics (e.g., number of feature gate checks)
    Increment(metricName string, value int, tags map[string]interface{}) error
    
    // Report gauge metrics (e.g., current number of active users)
    Gauge(metricName string, value float64, tags map[string]interface{}) error
    
    // Report distribution metrics (e.g., response times, payload sizes)
    Distribution(metricName string, value float64, tags map[string]interface{}) error
    
    // Control high cardinality tag filtering for performance
    ShouldEnableHighCardinalityForThisTag(tag string) bool
    
    // Clean up resources when shutting down
    Shutdown(ctx context.Context) error
}
```

### Key Features

- **Automatic Metric Prefixing**: All metrics are automatically prefixed with `statsig.sdk.`
- **Error Recovery**: The SDK handles panics and errors from your observability client gracefully
- **High Cardinality Filtering**: Control which tags are included to manage metric cardinality
- **Context Support**: All initialization and shutdown methods support Go context for timeout control
- **Thread Safety**: The SDK handles concurrent access to your observability client safely
